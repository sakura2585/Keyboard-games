# 鍵盤小遊戲 — 專案構成與延續說明

> 供新對話／新協作者快速接軌。專案根目錄：`H:\PY\鍵盤小遊戲`（本機路徑依環境而異）。

## 1. 專案目的

兒童向**英數與注音打字練習**：畫面顯示目標字元或單字，使用者按實體鍵盤對應鍵，累積分數與連擊。目前有**兩條實作線**：網頁版（功能較完整）與桌面版（tkinter）。

## 2. 目錄結構（精簡）

| 路徑 | 說明 |
|------|------|
| `web/index.html` | 網頁版單頁入口：模式與設定（兩列緊湊區塊）、能量條、可收合之英文詞彙種類（`<details>`）、模擬鍵盤、本局結束遮罩等 DOM |
| `web/style.css` | 版面、**盲打手指分區鍵盤色**、題目鍵外框、能量條、背景插圖變數、跑馬／三欄單字區、詞彙 topic 按鈕等 |
| `web/images/bg-illustration.svg` | 預設頁面背景插圖（可換檔或 `--page-bg-illustration: none` 關閉） |
| `web/app.js` | **網頁版全部遊戲邏輯**（千行級）：狀態、`nextRound`、`onKeyDown`、辭彙載入、偏好、`requestAnimationFrame` 能量遞減、TTS、**Web Audio 音效**（按鍵／結束／破紀錄）、`buildKeyboard`（`data-finger`、F／J 之 `id`） |
| `web/data/vocabulary.json` | 共用資料：`wordPool`（簡單英文單字）、`bopomofoWordPool`（注音拼字） |

| `requirements.txt` | 桌面版依賴（多為標準庫＋註記） |
| `README.md` | 使用者向：怎麼跑、辭彙格式、常見問題 |
| `release_notes_latest.md` | 發行／變更備註（若有） |

## 3. 網頁版 vs 桌面版（功能對照）

| 項目 | 網頁 `web/` | 桌面 `main.py` |
|------|-------------|----------------|
| 模式 | `letter`／`english`／`word`／`bopomofo`／`bopomofo_word` | 前四者；**無** `bopomofo_word` |
| 能量條、局終、難度遞減 | 有 | 無（桌面為較早的簡化版） |
| 英文單字三欄跑馬（已完成／目前／下一題） | 有 | 無 |
| `wordPool` 的 **`topic` 多選篩選** | 有（`state.wordTopicFilters`）；UI 為可收合面板，且僅在 `word` 模式顯示 | **無**（僅讀 `en` 成字串清單） |
| TTS／發音 | Web Speech API；**發音時機**見下節 | 無對等實作 |
| 偏好儲存 | `localStorage` 鍵 `typing-game-web` | `state/app_state.json` |

實作新功能時請先確認要同步哪一邊，避免預設「改 `app.js` 就等於桌面也變」。

## 4. 網頁版核心流程（`app.js`）

- **`state`**：集中保存 `mode`、`score`、`streak`、`expected`、`chars`、`pos`、英文單字相關、`energy`、`gameOver`、`drainPerSec`、`speakWhen`、`speakRate`、**`sfxKeyEnabled`**（按鍵音效）、`wordTopicFilters`、`wordTopicPanelOpen`（詞彙種類面板是否展開）等。
- **`boot()`**：`loadVocabulary()` → `buildWordTopicBar()` → `buildKeyboard()` → `nextRound()` 等初始化。
- **`nextRound()`**：依 `state.mode` 抽題或組字串；`word` 模式會 `applyWordItemToGameState`、`renderWordMarqueePanes`，必要時觸發「成為題目」發音（見下）。
- **`onKeyDown`**：答對／錯誤、能量增減、`word`／`bopomofo_word` 多字元進位；單字打完時可能 `wordMarqueeAdvanceToNext()` 或 `nextRound()`，並在「完成題目」發音設定下呼叫 TTS。
- **`promoteWordMarqueeNextToCurrent`**：跳過或進下一題時把「下一題」升為「目前」；**成為題目發音**在此與 `nextRound` 的 word 分支觸發。
- **能量**：`energyTick`（rAF）依 `getDrainPerSec()` 遞減；連擊達門檻會加重遞減（`STREAK_HITS_PER_DRAIN_PLUS_ONE`，常數見檔首）。
- **答對分數**：基礎仍為 `1 + min(連擊,5)`，再乘 **`getScoreRewardMultiplier()`**（依即時每秒遞減在 1～`SCORE_MULT_AT_MAX` 之間線性放大）；答錯不減分數。

## 5. 發音（僅網頁・簡單英文單字模式）

- **UI**：**「發音」**為主開關（checkbox）；勾選後才啟用自動發音，並以**下拉選單**擇一：**成為題目發音**（`onQuestion`）或**完成題目發音**（`onComplete`）。未勾選主開關時等同 **`off`**（不自動發音），下拉為停用狀態。
- **`state.speakWhen`**：仍為 `onQuestion` | `onComplete` | `off`，與 `savePrefs`／`localStorage` 一致。
- 舊版偏好僅 **`speakEnabled`** 時：`true` → `onComplete`，`false` → `off`。
- **`speakUtterance`**：實際呼叫 `speechSynthesis`；**「發音（目前題目）」按鈕**不受上述自動發音設定限制（手動播放）。

## 6. `localStorage`：`typing-game-web`

JSON 大致欄位（實際以 `savePrefs()` 為準）：

- `mode`：`letter` | `english` | `word` | `bopomofo` | `bopomofo_word`
- `speakWhen`：`onQuestion` | `onComplete` | `off`
- `speakRate`：數字（約 0.5–1.5）
- `showKeyboard`：boolean
- `drainPerSec`：起始每秒能量遞減（1–40；實際遞減含連擊加成上限見 `DRAIN_PER_SEC_MAX`）
- `wordTopicFilters`：`null`（全部）或 topic 字串陣列（與 `vocabulary.json` 的 `topic` 對應）
- `wordTopicPanelOpen`：boolean，英文詞彙種類區塊（`<details>`）是否展開
- `sfxKeyEnabled`：boolean，答對／答錯**按鍵短音效**（結束與破紀錄音不受此欄位影響）

載入時若詞庫尚未就緒，topic 篩選可能先暫存在 `pendingWordTopicKeys`，於 `finishWordTopicHydration()` 再套入 `state.wordTopicFilters`。

**本局結束**：畫面上提示可按 **Enter** 或按鈕再玩一局；顯示遮罩時會將焦點移至「再玩一局」，關閉時若原先焦點不在遮罩內則嘗試還原。

**詞彙列點擊**：`onWordTopicBarClick` 僅在啟動時對 `#wordTopicBar` 綁定**一次**；`buildWordTopicBar()` 只重建子節點，避免重複監聽。

**分數紀錄（另鍵）**：`typing-game-web-score-history` 為 JSON **`{ v: 2, byMode }`**：`byMode` 鍵為五種 `mode`（`letter`／`english`／`word`／`bopomofo`／`bopomofo_word`），各模式**獨立**一組 `{ top, last }`——`top` 至多 **10** 筆（排名見 `compareRecordsDesc`），`last` 為該模式**最後一局**。每筆紀錄仍含 `ts`、`mode`、`score`、`correct`、`wrong`、`streak`。舊版單一 `{ top, last }` 或陣列會在讀取時**依每筆的 `mode` 分桶**並寫回 v2。結束畫面與「紀錄表」可透過下拉 **紀錄模式** 切換檢視；**Esc**／**關閉** 行為不變。

## 7. 辭彙資料 `web/data/vocabulary.json`

- **`wordPool`**：`{ "en": "小寫英文", "zh": "中文", "topic"?: "…" }`。`topic` 供網頁種類篩選；缺省在正規化時可視程式邏輯補成「其他」等。
- **`bopomofoWordPool`**：`{ "han", "bop" }`，`bop` 需符合 `app.js` 內 `BOPOMOFO_KEYMAP` 鍵位。
- 載入失敗或陣列空時，退回 `app.js` 內建 `DEFAULT_WORD_POOL`／`DEFAULT_BOPOMOFO_WORD_POOL`。

**本機開發請用 HTTP 伺服器開網頁**（例如 `cd web && python -m http.server`），不要用 `file://` 直接開 `index.html`，否則 `fetch("data/vocabulary.json")` 常失敗。

## 8. 執行方式（摘要）

```powershell

# 網頁版
```

## 9. 延伸修改時的檢查清單

- 改出題／按鍵判定：幾乎都在 **`web/app.js`** 的 `nextRound`、`onKeyDown`、`promoteWordMarqueeNextToCurrent`。
- 改 UI 文案或 DOM：**`index.html`** + **`style.css`**。
- 改模擬鍵盤手指配色或題目鍵外框：**`style.css`**（`--finger-*`、`--key-target*`）；鍵位與手指對照表在 **`app.js`** 之 `FINGER_BY_PHYSICAL_KEY`。
- 改按鍵／局終／破紀錄音效：**`app.js`**（`playSfxKeyCorrect`／`playSfxGameOver` 等與 `getSfxAudioContext`）。
- 改英文詞彙種類列：重建按鈕在 **`buildWordTopicBar`**；點擊邏輯在 **`onWordTopicBarClick`**（監聽器勿在 `buildWordTopicBar` 內重複 `addEventListener`）。
- 改本局結束行為：**`endRun`**／**`beginFreshRun`**／**`startNewRun`**。
- 改資料格式：**`vocabulary.json`** + `normalizeWordPoolEntries`／`normalizeBopomofoWordPoolEntries`（在 `app.js`）。
- 若需與桌面一致：額外改 **`main.py`**（目前無 topic 篩選、無注音拼字、無能量／TTS）。
